# Каталог букетов через CMS — дизайн

Дата: 2026-06-30
Статус: утверждён к реализации

## Цель

Дать владельцу боутика возможность самостоятельно добавлять/редактировать/удалять
букеты в каталоге через визуальную админку, без правки HTML. Сайт остаётся чистой
статикой (no build step), хостинг — Cloudflare Pages.

Вне объёма этого захода: редактирование произвольных текстов (герой, контакты,
часы) — отдельный следующий шаг.

## Стек

- **Sveltia CMS** (лёгкий форк Decap), подключается с CDN — ноль сборки.
- **Бэкенд** — GitHub repo. Сохранение из админки = коммит в репозиторий.
- **Авторизация** — GitHub OAuth App + официальный воркер `sveltia-cms-auth`
  на Cloudflare Workers.
- **Деплой** — Cloudflare Pages, автодеплой по push в master (~1 мин).

## Модель данных

Единый файл `content/bouquets.json` — массив объектов. Один файл выбран потому,
что статичный хостинг не умеет листать директорию, а без build-шага собрать
индекс файлов-на-товар нечем. Заполнение остаётся удобным: Sveltia рисует список
с add/remove/reorder и загрузкой фото на каждый элемент.

Объект букета:

| поле          | тип      | виджет Sveltia | назначение                                   |
|---------------|----------|----------------|----------------------------------------------|
| `title`       | string   | string         | название («Утренний свет»)                   |
| `image`       | string   | image          | путь к фото, загрузка в `assets/images/`     |
| `price`       | number   | number         | цена, рендерится как `<price> Br`            |
| `description` | string   | text           | состав букета                                |
| `available`   | boolean  | boolean        | если `false` — карточка не рендерится         |

Начальное наполнение: 6 существующих захардкоженных карточек переносятся в
`bouquets.json` (title / image / price / description, `available: true`), чтобы
ничего не потерялось.

## Компоненты

### 1. Админка
- `admin/index.html` — страница, подключающая Sveltia с CDN.
- `admin/config.yml` — конфиг:
  - `backend`: github, ветка master, `base_url` = URL воркера авторизации.
  - `media_folder: assets/images`, `public_folder: assets/images`.
  - file-коллекция `bouquets` → файл `content/bouquets.json`, поле-список
    `items` с виджетами из таблицы выше.

### 2. Данные
- `content/bouquets.json` — единственный источник правды для каталога.

### 3. Рендер на фронте (`index.html`, в существующем `<script>`)
Маленький модуль каталога:
1. `fetch('content/bouquets.json')`.
2. Фильтрует `available !== false`.
3. Строит `<article class="bouquet-card reveal">` из тех же классов/токенов,
   что сейчас в разметке (media с `background-image`, body с `h3`/`p`/price).
   Цена → `${price} Br`.
4. Вставляет карточки в `.card-grid` (очистив статичный плейсхолдер).
5. **Переинициализация потребителей списка** (устраняет ручную синхронизацию):
   - лайтбокс — навесить обработчик клика на новые `.bouquet-card__media`;
   - `<select id="order-bouquet">` — пересобрать `<option>` из названий,
     **сохранив** первый disabled-плейсхолдер и последнюю опцию
     «Хочу индивидуальный заказ».
6. Повесить `.reveal`-элементы карточек в существующий IntersectionObserver
   (guard под `prefers-reduced-motion` сохраняется).

## Обработка ошибок

- Если `fetch` упал / JSON битый: в `.card-grid` остаётся статичный
  fallback-текст («Каталог временно недоступен, позвоните нам»), не белый экран.
  Селект и лайтбокс в этом случае работают по тому, что уже есть.

## Поток данных

```
владелец → сайт/admin/ → (GitHub OAuth через воркер) → правит список
   → Sveltia коммитит content/bouquets.json + фото в assets/images
   → push в master → Cloudflare Pages передеплой
   → посетитель открывает index.html → fetch bouquets.json → карточки
```

## Настройка (разовая, вне кода — инструкция пользователю)

1. Залить репозиторий на Cloudflare Pages (build command пустой, output = корень).
2. Создать GitHub OAuth App (callback на воркер).
3. Задеплоить `sveltia-cms-auth` воркер, прописать client id/secret.
4. Вписать `base_url` воркера в `admin/config.yml`.

## Тестирование

- Локально через `npm run dev`: `bouquets.json` отдаётся, карточки рендерятся,
  лайтбокс открывает фото, селект содержит все названия + спецопции, число
  карточек = числу `available` товаров.
- Ломаем JSON → видим fallback, страница не падает.
- Узкие вьюпорты ~360px / ~768px: сетка карточек корректна (как сейчас).

## Затрагиваемые файлы

- `index.html` — замена статичного списка карточек на контейнер + JS-рендер;
  правка опций селекта (динамическое заполнение).
- `content/bouquets.json` — новый, с переносом 6 карточек.
- `admin/index.html`, `admin/config.yml` — новые.
- НЕ трогаем: `experiments/`, `assets/css/*`,
  `docs/design-system.md` (новых стилей не требуется — переиспользуем классы).
