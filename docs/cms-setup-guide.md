# Руководство по настройке Sveltia CMS и авто-деплоя

В этом проекте используется **Sveltia CMS** для редактирования контента (каталога букетов). Админка работает напрямую с GitHub, а это значит, что при сохранении изменений она автоматически создает коммит в ветку `master` вашего репозитория.

Чтобы все заработало на реальном (боевом) сайте на хостинге (например, hoster.by), нужно выполнить два этапа: настроить авторизацию админки и настроить авто-доставку обновлений на хостинг.

---

## Этап 1. Настройка авторизации (OAuth Proxy)

Так как у нас статичный сайт, для безопасного подключения к GitHub нужен микро-сервис (прокси). Мы уже развернули его на Cloudflare Workers и вписали ссылку в `admin/config.yml` (`https://sveltia-cms-auth.kalininaugust.workers.dev`).

Вам осталось только «подружить» его с вашим аккаунтом GitHub:

1. Зайдите в GitHub: **[Settings → Developer Settings → OAuth Apps](https://github.com/settings/developers)**.
2. Откройте приложение `Flores CMS` (или создайте новое, если еще не создавали).
3. В поле **Homepage URL** впишите: `http://localhost:3000` (или реальный адрес сайта, когда он появится).
4. **ВАЖНО:** В поле **Authorization callback URL** обязательно вставьте эту ссылку:
   `https://sveltia-cms-auth.kalininaugust.workers.dev/callback`
5. Сохраните изменения. 

С этого момента при заходе на `ваш-сайт.com/admin/` кнопка «Login with GitHub» будет успешно вас авторизовывать.

---

## Этап 2. Настройка автоматического обновления сайта (GitHub Actions)

В корне проекта уже лежит скрытый файл скрипта: `.github/workflows/deploy.yml`. 
Он отвечает за то, чтобы при любом изменении в админке новые файлы 자동으로 улетали на ваш FTP (хостинг hoster.by).

Чтобы это заработало, GitHub должен знать логин и пароль от вашего FTP.

### Что нужно сделать:
1. Узнайте у вашего хостера (hoster.by) данные для подключения по FTP: **Сервер**, **Логин** и **Пароль**.
2. Зайдите на страницу вашего репозитория `flores` в GitHub.
3. Откройте вкладку **Settings** (Настройки).
4. В левом меню выберите **Secrets and variables** → **Actions**.
5. Нажмите зеленую кнопку **New repository secret** и добавьте ровно три секрета:
   - Имя: `FTP_SERVER` | Значение: *адрес вашего FTP сервера*
   - Имя: `FTP_USERNAME` | Значение: *ваш FTP логин*
   - Имя: `FTP_PASSWORD` | Значение: *ваш FTP пароль*

### Как проверить, что всё работает?
1. Зайдите в админку (`/admin/`), поменяйте что-нибудь (например, цену букета) и нажмите **Сохранить**.
2. Перейдите на GitHub в свой репозиторий на вкладку **Actions**. Вы увидите, что запустился процесс `Deploy to FTP`.
3. Подождите 1-2 минуты, пока кружок станет зеленым.
4. Откройте ваш сайт — изменения уже там!

---

*Примечание: Если на хостинге папка с сайтом называется не `public_html`, а как-то иначе (например, `www` или `htdocs`), не забудьте изменить параметр `server-dir: ./public_html/` в файле `.github/workflows/deploy.yml`.*
